Creating a text file for import to mono:di

In text documents for import to mono:di, rubrics and sung text must be organized in three tab separated columns.  Make sure that in the tab stop bar exactly two tab stops are set, one separating the first column with line labels from the second column with rubrics and sung text, one separating the second column from the third one with folio changes.  All columns must be separated by exactly one tab character.

  • line labels (trope element numbers, etc.) in column 1
  • sung text in column 2
  • rubrics in capital letters in column 2
  • folio changes in column 3
  • if folio breaks and line breaks from the source coincide with editorial line breaks, they need to be marked at the beginnings of the editorial lines, not at the endings
  • base chants should be in normal type, not capitals or small capitals
  • all elements receive their own line, even if they are short and will appear on the same line in the print edition
  • start folio belongs in the header data, on the line with Übersichtszeile. It must be entered manually as part of the file’s header data
  • do not import the Übersichtszeile—enter the information manually in the header data (except for the trope element numbers and letters for base chant cues, which will be automatically generated).

Managing files

When a user moves from a document to the management page, the document is still open and can be accessed by clicking “Document.”

To save a file to your computer, so that you can work without an internet connection,

  • go to the management page
  • press the blue arrow button next to the file name
  • document is locally available when button turns yellow

To work with perform the same command with multiple files,

  • click the boxes prior to the file names
  • select a “batch function” from the drop-down list at the top of the management screen

Preparing Firefox for off-line functions

  • Open Firefox window
  • select drop-down list under “Firefox”
  • select “Preferences”
  • select “Privacy”
  • under “History” select “Firefox will: ‘Use custom settings for history’ ”
  • UNCHECK box in front of “Clear history when Firefox closes”

Working from a USB drive

Documents that are locally available (marked with a yellow button in management view) are placed in a small database on your hard disk that is managed by the browser you are using. If you set up multiple user profiles on your computer or for your browser, an independent database will be used for each user profile.

If you saved your work while being online, you can log in from any computer and any supported browser to download your saved work from the server. However, it is also possible to take your locally stored data with you on a USB drive by using a portable browser that runs off a USB drive.

Portable browsers are available for Windows and Mac. They only work on one or the other, so the local data can only be taken from one Mac to another or from one Windows PC to another. Using a portable browser is only recommended if you are working on multiple computers with the same operating system and your internet connection is unreliable.

Excellent portable browser versions for Windows are supplied by portableapps.com. Portable Mac versions are generally not as well supported. Users of portable browsers are advised to check regularly if their browser version is still up to date for security reasons. If the browser does not auto-update itself, newer versions have to be downloaded manually.

Explanation of header fields

Above the transcription, fields are listed where information concerning the entire chant can be recorded. This information will enable certain search functions for users of CM digital.

Line 1

CM section no. – volume no.

for example, CM II-1

Line 2

source no.

each source in the CM volume will receive a number, based on the order it appears in the volume

source identifier

this is the abbreviation for the manuscript’s shelfmark (Provins, Bibliothèque municipale 12 (24) is recorded here as Pro 12)

Line 3

transcription number

This number appears in the print edition at the beginning of the transcription with a square around it. This is the number of the transcription within each source. (The first trope transcribed from Pro 12 will have a 1 in this field.)

genre

of the chant (trope = trop; offertory = off)

incipit

standardized spelling

feast

also abbreviated and standardized (Nativitas = Nat)

start folio

of the chant in the manuscript (f. 22v)

Line 4

All fields on this line are recorded from the text edition. If the text edition does not contain the information, the field can be left blank.

text edition incl. volume no

(for example, CT I)

text’s identifying number

only for text editions like Analecta hymnica, where each text receives its own number

base chant incipit

this field is for CT X and CT IX, which uses a base chant incipit to identify each element, and we cite exactly corresponding to the CT.

feast, service, base chant genre:

as they appear in CT, for example Nat III intr

Line 5

melody catalogue author, melody no., trope complex no.

these fields are for CM’s volumes of ordinary genres, which have catalogues of base chant melodies and trope incipits (such as Schildbach)

To transfer a volume to ngb, editors will need to save the files with names that will place the transcriptions in the volume order, for example:

1 Hodie cantandus est 

2 Puer natus est nobis (although the transcription number in the file is 1P)

Troubleshooting

Printing

Staffs become solid black rectangles

This problem is likely stemming from problems with the printer driver.  Try to increase the print resolution.  If you don't succeed, try creating a PDF with Chrome's built-in PDF print:

  • Open Chrome's print dialogue (Ctrl+P)
  • click the "Change" button
  • select "Save as PDF"
  • click "Save"
  • print the saved PDF using a PDF viewer
